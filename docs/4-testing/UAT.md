# User Acceptance Testing (UAT)

## Detail Pelaksanaan
*   **Nama Proyek:** Aplikasi Computer Based Test (CBT) Enterprise
*   **Tanggal Pengujian:** [Isi Tanggal]
*   **Lokasi/Metode:** [Online / Offline]

## Tujuan
Memastikan aplikasi yang dikembangkan telah memenuhi kebutuhan institusi (Sesuai dengan SRS) dan siap digunakan secara massal oleh Guru dan Siswa.

## Kriteria Penerimaan
Fitur-fitur utama berikut harus dinyatakan **LULUS** (PASS) oleh klien/pengguna perwakilan:

| ID | Fitur | Deskripsi | Status (Pass/Fail) | Catatan |
|---|---|---|---|---|
| UAT-1 | Manajemen User | Admin dapat menambah/menghapus Siswa dan Guru. | | |
| UAT-2 | Pembuatan Soal | Guru dapat mengetik soal (Rich Text) atau impor dari Word. | | |
| UAT-3 | Mengerjakan Ujian | Siswa dapat login, memasukkan token, dan mengerjakan ujian tanpa *lag*. | | |
| UAT-4 | Live Proctoring | Guru dapat melihat daftar siswa yang "LIVE" dan melihat peringatan jika siswa pindah tab. | | |
| UAT-5 | Grading & Export | Guru dapat memberikan nilai Essay dan mengunduh rekap `.xlsx`. | | |
